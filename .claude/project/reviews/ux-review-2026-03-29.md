# UX 리뷰 — 메인→검색 페이지 전환

> 일자: 2026-03-29
> 대상: 메인(/) → 검색(/bookmarks) 전환 흐름
> 기준: Nielsen 10 Heuristics, Apple HIG, Dieter Rams, WCAG 2.1 AA

---

## Critical (즉시 수정)

### [1] 검색창 포커스만으로 페이지 전환

- `apps/web/src/components/layout/Header.tsx:112-117`

```ts
const handleSearchFocus = () => {
  if (pathname !== "/bookmarks") {
    router.push(`/bookmarks...`); // 포커스만 해도 페이지 교체
  }
};
```

**원칙:** Apple HIG 직접 조작 원칙 위반. 사용자가 의도하지 않은 페이지 이동.
**영향:** 검색창 클릭(탭 이동 포함)만 해도 화면이 바뀜. 모바일에서 특히 심각.
**수정:** `onFocus`에서 `router.push` 제거. `onChange` + debounce 또는 엔터 제출 시점에만 이동.

---

### [2] 키워드 카운트가 시맨틱 결과 제외

- `apps/web/src/app/bookmarks/BookmarksContent.tsx:209`

```ts
<p>"${keywordFiltered.length}개의 북마크"</p>
// 아래 시맨틱 결과는 카운트에 미포함
```

**원칙:** Nielsen #1 시스템 상태 가시성 위반.
**영향:** "3개의 북마크"라고 표시됐는데 스크롤하면 AI 결과가 더 나옴. 카운트 보고 이탈 가능.
**수정:** "3개 키워드 일치 + AI 결과 N개" 형태로 분리하거나 총합 카운트 표시.

## High (신뢰도·편의성 저하)

### [3] 홈 카드 태그 클릭이 기존 필터 초기화

- `apps/web/src/app/_home-content.tsx:37`

```ts
const handleTagClick = (tag: string) => {
  router.push(`/bookmarks?tag=${encodeURIComponent(tag)}`);
  // 기존 searchParams 태그 누적 없음
};
```

**영향:** Header FilterBar에서 `#React` 선택 후 홈 카드의 `#TypeScript` 클릭 시 기존 태그 초기화.
**수정:** 기존 `searchParams` 태그에 누적하도록 `Header.tsx`의 `buildBookmarksUrl` 헬퍼 재사용.

---

### [4] 태그 변경마다 시맨틱 검색 재실행

- `apps/web/src/app/bookmarks/BookmarksContent.tsx:62-97`

```ts
useEffect(() => {
  // ...
}, [query, selectedTags]); // selectedTags 포함 → 태그 토글마다 API 재호출
```

**영향:** 태그 필터 토글할 때마다 임베딩 API 재호출 → 느린 응답 + 불필요한 비용.
**수정:** deps를 `[query]`만으로 한정. 태그 필터는 반환된 결과를 클라이언트에서 필터링.

---

### [5] 최근 북마크 없으면 슬라이더 사라짐

- `apps/web/src/widgets/bookmark/RecentBookmarkSlider.tsx`

24시간 내 항목 없으면 `return null` → 홈에서 검색 진입 경로가 검색창뿐이 됨.
**수정:** "오늘 저장한 북마크가 없어요" + 북마크 추가 CTA 빈 상태 UI로 교체.

## 잘된 부분

- `buildBookmarksUrl` + `useSearchParams().getAll("tag")`로 URL 파라미터 기반 상태 관리 구조 탄탄함 (뒤로가기 정상 동작 보장)
- `BookmarkCard`의 AI 처리 3단계 시각화 (크롤링→AI처리→완료) 명확
- `SemanticResultSection`의 비회원 게이트 — 이유 + 해결책 동시 제시
