# QA 리뷰 결과 — 메인화면(/) 북마크 기능

> 분석 일자: 2026-03-29
> 분석 대상: 메인화면(/) 북마크 목록 조회, 추가, 최근 북마크 슬라이더, 상세 패널
> 기준: Google Web Vitals, Nielsen 10 Heuristics, WCAG 2.1 AA

---

### Critical (즉시 수정 — 사용자가 바로 인지)

- [AddBookmarkOverlay.tsx:154] ESC 닫기 없음 + body scroll lock 없음
  재현: 모달 열린 상태에서 ESC 키 무반응. 모바일 backdrop 뒤 페이지 스크롤됨 (iOS Safari 심각)
  영향: 키보드 사용자 전원 (WCAG 2.1.2). 모바일 사용자 전원
  수정: keydown 이벤트로 ESC → handleClose(). document.body.style.overflow = 'hidden' 적용

- [AddBookmarkOverlay.tsx:66-74] 비-한도 에러 발생 시 사용자 피드백 전무
  재현: 네트워크 오류, Supabase 장애 등에서 console.error만 실행. 사용자에게 아무 메시지 없음
  영향: 저장 실패 시 모든 사용자. 중복 시도로 이어짐
  수정: catch 블록에서 toast 노출

---

### High (출시 전 수정 — 어색함/불편함)

- [_home-content.tsx:16] 로딩 중 skeleton 없이 empty state 즉시 노출 (CLS)
  재현: 최초 진입 시 "북마크를 추가하세요." 먼저 표시 후 카드들이 갑자기 등장
  수정: isLoading 상태에서 skeleton grid 렌더링

- [BookmarkDetailPanel.tsx:57-59] 저장 실패 시 피드백 없음 + 편집 상태 강제 종료
  재현: 저장 실패 시 console.error만, isEditing이 false로 전환되어 저장된 것처럼 보임
  수정: catch에서 toast 노출. setIsEditing(false) 제거하여 편집 상태 유지

- [BookmarkDetailPanel.tsx:83-87] role="dialog", aria-modal, aria-hidden 없음
  수정: role="dialog" aria-modal="true" aria-label="북마크 상세" aria-hidden={!isOpen} 추가

- [AddBookmarkOverlay.tsx] 포커스 트랩 없음
  재현: Tab 키로 배경 콘텐츠로 포커스 빠져나감 (WCAG 2.4.3)
  수정: focus-trap 직접 구현 또는 @radix-ui/react-dialog 활용

- [BookmarkCard.tsx:53] new URL(url).hostname 예외 처리 없음
  재현: javascript:void(0), 공백 등 비표준 URL에서 TypeError → 카드 렌더링 크래시
  수정: try/catch로 감싸고 fallback 처리

- [AddBookmarkOverlay.tsx:147-150] 백그라운드 파이프라인 실패 시 모달 닫힌 후 무통보
  재현: 크롤링/AI 분석 실패 시 카드 상태는 failed지만 사용자 알림 없음
  수정: 파이프라인 실패 catch에서 toast 노출
