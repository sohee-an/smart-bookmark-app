# UX/UI 리뷰 결과

### Critical (즉시 수정)

**[C-1] 저장 버튼 라이트/다크 모드 색상 불일치**

- `AddBookmarkOverlay.tsx:200`, `BookmarkDetailPanel.tsx:258`
- 라이트 모드에서 주 CTA가 `bg-zinc-900`(검정), 다크모드에서 `dark:bg-brand-primary`(파란색)로 모드마다 완전히 다른 시각 언어 사용. 헤더 추가 버튼은 항상 `bg-brand-primary`라 세 버튼이 서로 다른 스타일.
- **수정:** 주 행동 버튼을 라이트/다크 모두 `bg-brand-primary text-white`로 통일.

**[C-2] 삭제 확인 UI에서 취소/삭제 버튼 위치 역전**

- `BookmarkDetailPanel.tsx:277~295`
- 왼쪽 "취소"가 `bg-zinc-900`(강조), 오른쪽 "삭제"가 흰 배경 빨간 글씨(약한 강조). 시각적으로 더 강한 버튼이 "취소"여서 오조작 유발 위험.
- **수정:** 취소를 secondary 스타일, 삭제를 destructive(강조) 스타일로 배치.

**[C-3] 모바일에서 상세 패널 닫기 제스처 없음**

- `BookmarkDetailPanel.tsx:122~125`
- 모바일에서 패널이 전체 화면을 덮으므로 배경 dim을 탭할 수 없고, 스와이프 제스처도 없어 닫기 버튼이 유일한 탈출 경로.
- **수정:** 우측 스와이프 제스처 또는 상단 드래그 핸들 추가.

### High (우선 수정)

**[H-1]** 아바타 드롭다운 `group-hover:block` — 모바일 터치에서 접근 불가 (`Header.tsx:214`) → `onClick` 토글 방식으로 변경

**[H-2]** 검색창 포커스 시 즉시 페이지 이동 (`Header.tsx:112~117`) → 포커스가 아닌 실제 검색어 입력/제출 시에만 이동

**[H-3]** `crawl_failed` 상태 카드에서 태그 영역 완전 숨김으로 그리드 높이 불일치 (`BookmarkCard.tsx:157`) → 카드에 `min-h` 지정

**[H-4]** `RecentBookmarkSlider` 빈 상태 시 섹션 자체가 사라짐 (`RecentBookmarkSlider.tsx:25~27`) → 신규 사용자 온보딩 프롬프트로 대체

**[H-5]** 모바일 헤더 이중 구조로 sticky 영역 120px+ 점유 (`Header.tsx:248~270`) → 검색을 별도 drawer로 분리

**[H-6]** 저장 성공 후 모달 닫기 시 `url`, `memo` 상태 미초기화 (`AddBookmarkOverlay.tsx:41~44`) → `handleClose`에 상태 초기화 추가

### 잘된 부분

1. **로딩 상태 4단계 세분화** — `crawling/processing/failed/crawl_failed` 각각 다른 UI, skeleton과 blur+grayscale 구분 탁월
2. **슬라이더 네이티브 스냅 스크롤** — `snap-x snap-mandatory` + 스크롤바 제거, 모바일 경험 자연스러움
3. **삭제 2단계 확인 + 배경 dimming** — `isConfirmingDelete`로 실수 방지, 배경 `opacity-40`으로 맥락 유지
4. **비회원 한도 UX 인라인 처리** — 별도 페이지 이동 없이 모달 내 상태 전환으로 동선 단절 없음
5. **Escape 키 지원** — 키보드 접근성 기준 충족
6. **URL 복사 2초 피드백** — `isCopied` 상태로 조용하지만 명확한 완료 신호
7. **다크모드 `dark:` prefix 일관 적용** — 대부분의 색상 클래스에 짝으로 대응
8. **에러 인라인 클리어** — URL 수정 시작과 동시에 에러 메시지 즉시 제거
