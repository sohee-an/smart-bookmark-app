## 실행 순서

1. Notion 세부 태스크 DB 전체 조회
2. 태스크가 없으면 "진행할 태스크가 없습니다" 출력 후 종료
3. **시나리오 없는 태스크 먼저 감지** → 테스트 시나리오 작성
4. 나머지 태스크는 현재 속성 상태를 보고 해당 단계 실행

## 새 태스크 감지 규칙

아래 조건 중 하나라도 해당하면 "시나리오 미작성 태스크"로 간주한다:

- 상태가 비어 있음 (NULL)
- 페이지 본문에 `## 테스트 시나리오` 섹션이 없음

감지 시 동작:

1. 태스크명 기준으로 완료 기준 추론
2. 관련 코드 파악 (필요 시 실제 파일 읽기)
3. `## 테스트 시나리오` 섹션 작성 (Given/When/Then)
4. 상태를 `테스트 시나리오`로 설정

## 상태 흐름

```
테스트 시나리오 (디폴트, 생성 즉시 AI 작성)
    → (시나리오검수=true)
    → 방향성 + AI: 개발구현 계획 작성
    → (개발구현방향=true)
    → 개발중 → 테스트중 → 완료
```

## 단계별 진행 조건

```
상태=테스트 시나리오, 시나리오검수=false, 시나리오 피드백=없음
    → 대기 (사용자가 시나리오 검토 후 시나리오검수 체크 대기)

상태=테스트 시나리오, 시나리오검수=false, 시나리오 피드백=있음
    → 피드백 반영하여 ## 테스트 시나리오 재작성
    → ## 시나리오 피드백 섹션 비움 (반영 완료 표시)
    → 다시 대기

시나리오검수=true, 상태=테스트 시나리오
    → 상태를 방향성으로 변경
    → Step 1. 개발구현 계획 작성 (노션 페이지에 추가)
    → 개발구현방향 체크 대기

상태=방향성, 개발구현방향=false, 방향 피드백=없음
    → 대기 (사용자가 구현 계획 확인 중)

상태=방향성, 개발구현방향=false, 방향 피드백=있음
    → 피드백 반영하여 ## 구현 계획 재작성
    → ## 방향 피드백 섹션 비움 (반영 완료 표시)
    → 다시 대기

상태=방향성, 개발구현방향=true
    → Step 2. 구현
```

**체크박스 확인 없이 다음 단계 진행 절대 금지.**

---

## 태스크 생성 시 즉시 작성 (new-task 실행 시)

태스크 페이지 생성과 동시에 아래 섹션을 작성한다.

### `## 테스트 시나리오`

- 사용자가 눈으로 확인할 수 있는 동작 기준으로 작성
- Given / When / Then 형식
- 모든 케이스 (정상, 오류, 엣지) 포함

### `## 시나리오 피드백`

빈 섹션 — 시나리오가 마음에 들지 않을 때 여기에 자유롭게 의견 작성
AI가 반영 후 ## 테스트 시나리오 재작성 + 이 섹션 비움

---

## Step 1. 개발구현 계획 작성 (시나리오검수=true 감지 후)

상태를 `방향성`으로 변경 후 아래 섹션을 Notion 페이지에 추가한다.

### `## 관련 코드 수집`

- 구현에 필요한 기존 파일을 실제로 읽어서 파악
- 연관 파일 경로, 재사용 가능한 타입/함수/컴포넌트, 새로 만들 파일 목록

### `## 구현 계획`

- 테스트 시나리오를 기준으로 구현 방향 기술
- 수정할 파일, 핵심 로직 흐름, 코드 스니펫 포함

### `## 방향 피드백`

빈 섹션 — 구현 계획이 마음에 들지 않을 때 여기에 자유롭게 의견 작성
AI가 반영 후 ## 구현 계획 재작성 + 이 섹션 비움

**진행 조건: `개발구현방향=true` 확인 후에만 Step 2로**

---

## Step 2. 구현

1. `## 방향 피드백` 섹션 확인
   - 내용 있음 → 피드백 반영해서 구현
   - 내용 없음 → `## 구현 계획` 대로 구현
2. **구현 완료 기준: 유닛 테스트 통과 AND 테스트 시나리오의 동작이 실제로 구현됨**
3. Notion 상태 → `개발중`
4. 유닛 테스트 실행 → Step 3으로

## Step 3. 테스트 & 완료 처리

- 유닛 테스트 실행
- 전부 통과 → Notion 상태 `완료`
- 실패 → `.claude/skills/task-pipeline/ESCALATION.md` 참고
