# Notion 상태 전환 규칙

## Notion DB 속성 목록

### 세부 태스크 DB 필수 속성

| 속성명         | 타입     | 설명                                                   |
| -------------- | -------- | ------------------------------------------------------ |
| `상태`         | Select   | 방향성/개발중/테스트중/완료/재검토필요/재개발/검수요청 |
| `검수`         | Checkbox | 방향성 단계 검수 완료 여부                             |
| `시나리오검수` | Checkbox | 시나리오 검수 완료 여부                                |
| `선택방향`     | Select   | A / B (구현 방향 선택)                                 |
| `Red확인`      | Checkbox | Red 단계 확인 완료 여부                                |
| `실패횟수`     | Number   | 테스트 연속 실패 횟수 (5회 시 검수요청)                |
| `관련파일`     | Text     | 기획서+아키텍처에서 탐색된 관련 파일 목록              |

### 체크박스 진행 조건 요약

```
방향성 단계     →  검수=true        →  시나리오 작성 시작
시나리오 작성   →  시나리오검수=true →  테스트 코드 작성
방향 제시       →  선택방향=A or B  →  Red 확인
Red 확인        →  Red확인=true     →  구현 시작
```

### 큰 태스크 DB 필수 속성

| 속성명 | 타입   | 설명                                   |
| ------ | ------ | -------------------------------------- |
| `상태` | Select | 진행중/완료/재검토필요/재개발/검수요청 |

## 세부 태스크 상태값

| 상태         | 의미                        | 변경 주체    |
| ------------ | --------------------------- | ------------ |
| `방향성`     | 태스크 분해 완료, 검수 대기 | AI 자동      |
| `개발중`     | 구현 진행 중                | AI 자동      |
| `테스트중`   | 테스트 실행 중              | AI 자동      |
| `완료`       | 테스트 전부 통과            | AI 자동      |
| `재검토필요` | 기획 변경 영향 감지         | AI 자동      |
| `재개발`     | 재검토 후 수정 확정         | 사람 판단 후 |
| `검수요청`   | 5회 이상 실패               | AI 자동      |

## 큰 태스크 상태값

| 상태         | 의미                           | 변경 조건    |
| ------------ | ------------------------------ | ------------ |
| `진행중`     | 세부 태스크 개발 중            | 기본값       |
| `완료`       | 모든 세부 태스크 완료          | AI 자동      |
| `재검토필요` | 세부 태스크 중 재검토필요 존재 | AI 자동      |
| `재개발`     | 재검토 후 수정 확정            | 사람 판단 후 |

## 상태 전환 흐름

```
방향성 → (검수=true) → 개발중 → 테스트중
                                    │
                          ┌─────────┴──────────┐
                          ▼                    ▼
                       전부 통과           실패 발생
                          │                    │
                          ▼                    ▼
                        완료            AI 자가 수정
                          │             (횟수 카운트)
                   기획 변경 감지              │
                          │           5회 이상 실패
                          ▼                    │
                     재검토필요            검수요청
                          │
               ┌──────────┴──────────┐
               ▼                     ▼
          방향 맞음              방향 다름
               │                     │
               ▼                     ▼
            재개발            댓글로 수정 지시
                                      │
                                   방향성
                               (사람 재검수)
```

## 재검토필요가 붙는 경우

상태 무관하게 기획 변경 영향을 받으면 재검토필요로 변경:

```
완료된 태스크    → 이미 만든 게 달라져야 할 수도 있음
개발중인 태스크  → 지금 만들고 있는 방향이 틀렸을 수도 있음
방향성 태스크    → 아직 시작 전인데 기획이 바뀐 경우
```
