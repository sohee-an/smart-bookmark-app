사용자가 말한 내용을 GitHub 이슈로 만들어주세요.

## 1단계 — 이슈 분류

$ARGUMENTS 내용을 읽고 아래 기준으로 분류합니다.

| type    | 기준                    |
| ------- | ----------------------- |
| `bug`   | 오류, 안 됨, 깨짐, 에러 |
| `feat`  | 새 기능, 추가, 만들어줘 |
| `chore` | 설정, 정리, 리팩토링    |
| `docs`  | 문서, README            |

## 2단계 — 이슈 제목 작성

형식: `[type] 한 줄 요약`

예시:

- `[feat] 북마크 검색 기능 추가`
- `[bug] 로그아웃 후 게스트 데이터 남아있는 문제`
- `[chore] GitHub Actions CI 설정`

## 3단계 — 이슈 본문 작성

아래 템플릿으로 작성합니다.

```markdown
## 개요

<!-- 한 줄 요약 -->

## 배경 / 문제 상황

<!-- 왜 필요한가, 현재 어떤 문제가 있는가 -->

## 작업 범위

<!-- 구체적으로 뭘 해야 하는지 -->

- [ ]
- [ ]

## 완료 기준

<!-- 이슈가 닫혀도 되는 시점 -->

- [ ]
```

$ARGUMENTS 내용을 기반으로 최대한 채워주세요.
모르는 부분은 비워두고, 아는 부분만 채웁니다.

## 4단계 — 확인 후 생성

작성한 제목과 본문을 먼저 보여주고, 사용자 승인을 받은 뒤 이슈를 생성합니다.

```bash
gh issue create --title "제목" --body "본문"
```

생성 후 이슈 URL을 알려주세요.
