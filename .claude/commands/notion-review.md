## 실행 순서

1. Notion 세부 태스크 DB에서 진행할 태스크 탐색
2. 태스크가 없으면 "진행할 태스크가 없습니다" 출력 후 종료
3. 태스크의 현재 속성 상태를 보고 해당 단계 실행

## 단계별 진행 조건

```
검수=true, 시나리오검수=false
    → Step 1. 테스트 시나리오 + 관련 코드 수집 + 유닛 테스트 작성

시나리오검수=true, 개발구현방향=false
    → 대기 (사용자가 시나리오 확인 후 개발구현방향 체크 또는 방향 피드백 작성 중)

시나리오검수=true, 개발구현방향=true
    → Step 2. 구현
```

**체크박스 확인 없이 다음 단계 진행 절대 금지.**

---

## Step 1. 테스트 시나리오 + 관련 코드 수집 + 유닛 테스트 작성

Notion 태스크 페이지 본문에 아래 섹션을 순서대로 작성한다.

### `## 테스트 시나리오 작성`

- 사용자가 눈으로 확인할 수 있는 동작 기준으로 작성
- Given / When / Then 형식
- 모든 케이스 (정상, 오류, 엣지) 포함

### `## 관련 코드 수집`

- 구현에 필요한 기존 파일을 실제로 읽어서 파악
- 아래 항목을 정리:
  - 연관 파일 경로 및 현재 구조 요약
  - 재사용 가능한 타입 / 함수 / 컴포넌트
  - 새로 만들어야 할 파일 목록

### `## 유닛 테스트 코드`

- 순수 함수 / 로직 레이어만 테스트 (UI 통합 테스트 X)
- 테스트 시나리오의 동작을 기능 단위로 분해해서 작성
- 실제 테스트 파일도 코드베이스에 생성

### `## 구현 계획`

- 테스트 시나리오를 기준으로 구현 방향 기술
- 유닛 테스트 통과만이 완료 기준이 아님 — 시나리오의 동작이 실제로 구현되어야 완료

### `## 방향 피드백`

빈 섹션 — 계획이 맘에 들지 않을 때 여기에 직접 의견 작성

**진행 조건: `시나리오검수=true` + `개발구현방향=true` 확인 후에만 Step 2로**

---

## Step 2. 구현

1. `## 방향 피드백` 섹션 확인
   - 내용 있음 → 피드백 반영해서 구현
   - 내용 없음 → `## 구현 계획` 대로 구현
2. **구현 완료 기준: 유닛 테스트 통과 AND 테스트 시나리오의 동작이 실제로 구현됨**
3. Notion 상태 → `개발중`
4. 유닛 테스트 실행 → Step 3으로

## Step 3. 테스트 & 완료 처리

- 유닛 테스트 실행
- 전부 통과 → Notion 상태 `완료`
- 실패 → `.claude/skills/task-pipeline/ESCALATION.md` 참고
