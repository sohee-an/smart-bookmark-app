# OWASP Security Check 셋업 가이드

이 문서를 Claude Code CLI에게 넘기면 단계별로 셋업을 진행한다.

---

## 개요

OWASP Top 10 기반 보안 체크를 3단계로 자동화한다.

- **1단계 pre-push hook** — AI 없이 정적 패턴 체크 (빠름, 토큰 0)
- **2단계 슬래시 커맨드** — `/owasp-check` 실행 시 AI가 변경 파일 분석
- **3단계 GitLab CI** — MR 오픈 시 diff만 AI에게 전달 (토큰 최소화)

적용 프로젝트:

- `smart-bookmark` (Next.js / TypeScript)
- `MMIS` (Java / Spring)

### `~/.owasp-check/checklist.md`

OWASP Top 10 2021 기준 체크 항목. 공부하면서 직접 채워넣을 예정이므로 지금은 템플릿만 생성한다.

```markdown
# OWASP Top 10 체크리스트

## A01 - Broken Access Control

- [ ] 인증 없이 접근 가능한 엔드포인트가 있는가?
- [ ] 다른 사용자의 리소스에 ID만 바꿔서 접근 가능한가? (IDOR)
- [ ] 관리자 페이지가 role 검증 없이 열려있는가?

## A02 - Cryptographic Failures

- [ ] 비밀번호를 평문으로 저장하고 있는가?
- [ ] HTTP로 민감한 데이터를 전송하는가?
- [ ] 시크릿 키가 코드에 하드코딩 돼 있는가?

## A03 - Injection

- [ ] 사용자 입력이 SQL 쿼리에 직접 들어가는가?
- [ ] 사용자 입력이 shell 명령어에 사용되는가?
- [ ] ORM 없이 raw query를 사용하는 경우 파라미터 바인딩을 하는가?

## A04 - Insecure Design

- [ ] 재시도 제한 없이 로그인/OTP 시도가 가능한가?
- [ ] 비즈니스 로직을 클라이언트에서만 검증하는가?

## A05 - Security Misconfiguration

- [ ] 디버그 모드가 프로덕션에서 켜져 있는가?
- [ ] 불필요한 포트/서비스가 열려있는가?
- [ ] 에러 메시지에 스택 트레이스가 그대로 노출되는가?

## A06 - Vulnerable Components

- [ ] 알려진 취약점이 있는 라이브러리 버전을 사용하는가?
- [ ] 의존성 버전이 고정(lock)돼 있는가?

## A07 - Auth Failures

- [ ] 세션 토큰이 로그아웃 후 무효화되는가?
- [ ] JWT 서명 검증을 하는가?
- [ ] 브루트포스 방어(rate limit)가 있는가?

## A08 - Data Integrity Failures

- [ ] 외부에서 온 데이터를 검증 없이 역직렬화하는가?
- [ ] CI/CD 파이프라인에 무결성 검사가 없는가?

## A09 - SSRF

- [ ] 사용자 입력 URL을 서버가 직접 요청하는 기능이 있는가?
- [ ] 허용 호스트 화이트리스트가 있는가?
- [ ] 사설 IP 대역(127.x, 10.x, 169.254.x)을 차단하는가?

## A10 - Logging Failures

- [ ] 인증 실패 이벤트를 로깅하는가?
- [ ] 로그에 비밀번호/토큰이 찍히는가?
- [ ] 공격 시도를 감지하고 알림을 보내는가?
```

## Task 2. pre-push hook 생성

두 프로젝트 모두에 동일하게 적용한다.

### 생성 위치

- `smart-bookmark/.githooks/pre-push`
- `MMIS/.githooks/pre-push`

### 내용

```bash
#!/bin/bash

echo "🔍 OWASP 정적 체크 실행 중..."

FAILED=0

# 1. 하드코딩된 시크릿 키 패턴 감지
SECRET_PATTERNS=(
  "AWS_SECRET_ACCESS_KEY\s*="
  "aws_secret_access_key\s*="
  "password\s*=\s*['\"][^'\"]{4,}"
  "api_key\s*=\s*['\"][^'\"]{4,}"
  "secret\s*=\s*['\"][^'\"]{4,}"
  "PRIVATE_KEY\s*="
)

for pattern in "${SECRET_PATTERNS[@]}"; do
  result=$(git diff --cached --name-only | xargs grep -rn --include="*.ts" --include="*.js" --include="*.java" --include="*.properties" -E "$pattern" 2>/dev/null)
  if [ -n "$result" ]; then
    echo "❌ [A02] 하드코딩된 시크릿 의심 패턴 발견:"
    echo "$result"
    FAILED=1
  fi
done

# 2. HTTP 하드코딩 (HTTPS 아닌 것)
HTTP_RESULT=$(git diff --cached --name-only | xargs grep -rn --include="*.ts" --include="*.js" --include="*.java" -E "http://(?!localhost|127\.0\.0\.1)" 2>/dev/null)
if [ -n "$HTTP_RESULT" ]; then
  echo "⚠️  [A02] HTTP URL 하드코딩 발견 (HTTPS 사용 권장):"
  echo "$HTTP_RESULT"
fi

# 3. console.log / System.out.println 에 민감 단어 포함 여부
SENSITIVE_LOG=$(git diff --cached --name-only | xargs grep -rn --include="*.ts" --include="*.js" -E "console\.(log|error|warn).*?(password|token|secret|key)" 2>/dev/null)
if [ -n "$SENSITIVE_LOG" ]; then
  echo "❌ [A10] 로그에 민감 정보 포함 의심:"
  echo "$SENSITIVE_LOG"
  FAILED=1
fi

JAVA_SENSITIVE_LOG=$(git diff --cached --name-only | xargs grep -rn --include="*.java" -E "System\.out\.print.*?(password|token|secret|key)" 2>/dev/null)
if [ -n "$JAVA_SENSITIVE_LOG" ]; then
  echo "❌ [A10] 로그에 민감 정보 포함 의심 (Java):"
  echo "$JAVA_SENSITIVE_LOG"
  FAILED=1
fi

if [ $FAILED -eq 1 ]; then
  echo ""
  echo "🚫 보안 체크 실패. push가 중단됐습니다."
  echo "   문제를 수정하거나 확인 후 다시 시도하세요."
  exit 1
fi

echo "✅ 정적 보안 체크 통과"
exit 0
```

### hook 등록 명령어

각 프로젝트 루트에서 실행한다.


### `smart-bookmark/.claude/commands/owasp-check.md`

```markdown
# /owasp-check

git diff로 변경된 파일 목록을 추출하고, 각 파일을 OWASP Top 10 기준으로 분석한다.

## 실행 순서

1. `git diff HEAD --name-only`로 변경된 파일 목록 추출
2. `~/.owasp-check/ignore-patterns.txt`에 해당하는 파일 제외
3. 남은 파일들의 내용을 읽어서 `~/.owasp-check/prompt.md` 기준으로 분석
4. 취약점 발견 시 항목(A01~A10), 파일 위치, 수정 방향 출력
5. 이상 없으면 "✅ OWASP 체크 이상 없음" 출력

## 분석 기준 파일

- 주요 분석 대상: `src/` 하위 `.ts`, `.tsx` 파일
- API 라우트 우선 분석: `src/app/api/`, `src/pages/api/`
- 외부 요청 코드 우선 분석: `fetch`, `axios`, `http` 포함 파일

## 토큰 절약 규칙

- 변경되지 않은 파일은 분석하지 않는다
- 테스트 파일(_.test.ts, _.spec.ts)은 제외한다
- 파일당 분석은 핵심 로직 부분만 추출해서 전달한다
```

## Task 4. GitLab CI 설정 (선택)

MR 오픈 시 자동으로 diff만 AI에게 전달한다.

### 생성 위치

- 각 프로젝트 루트의 `.gitlab-ci.yml`에 아래 job 추가

```yaml
owasp-ai-check:
  stage: test
  only:
    - merge_requests
  script:
    - |
      # 변경된 파일만 추출 (토큰 최소화 핵심)
      CHANGED_FILES=$(git diff origin/$CI_MERGE_REQUEST_TARGET_BRANCH_NAME --name-only \
        | grep -E '\.(ts|tsx|java)$' \
        | grep -v -f ~/.owasp-check/ignore-patterns.txt)

      if [ -z "$CHANGED_FILES" ]; then
        echo "분석할 파일 없음"
        exit 0
      fi

      echo "분석 대상 파일:"
      echo "$CHANGED_FILES"

      # 실제 AI 호출은 Claude Code CLI 또는 Anthropic API로 연결
      # 구체적인 연동 방식은 프로젝트 환경에 맞게 설정
```

> CI 연동은 환경 변수(ANTHROPIC_API_KEY)가 GitLab CI/CD Variables에 등록돼 있어야 한다.
