# 024 — 익스텐션 AI 북마크 정리 최적화

## 배경

AI 북마크 정리 기능에서 350~400개 북마크를 정리할 때 약 3분 소요.
원인은 모든 북마크를 하나의 거대한 프롬프트로 Gemini에 단일 요청하는 구조였음.

---

## 문제 1: 단일 요청 → 느린 응답

### 기존 방식

```
items 400개 → 프롬프트 1개 → Gemini 단일 호출 → 응답 대기 3분
```

### 해결: 청크 분할 + 병렬 처리

```
items 400개 → 100개씩 4청크 → Promise.all 병렬 호출 → 결과 머지
```

- `CHUNK_SIZE = 100`으로 분할
- `Promise.all`로 모든 청크 동시 요청
- 응답받은 카테고리를 이름 기준으로 머지 (같은 이름이면 items 합산)
- 3분 → 약 20~30초로 단축

---

## 문제 2: 프롬프트 토큰 낭비

### 기존 방식

```
1. 제목: React 성능 최적화를 위한 완전한 가이드 | URL: https://www.medium.com/...
```

전체 URL + 전체 제목을 그대로 전송 → 토큰 과다 소비

### 해결: 인덱스 + 도메인 + 제목 40자 축약

```
1. React 성능 최적화를 위한 완전 | medium.com
```

- URL → 도메인만 추출 (`new URL().hostname`)
- 제목 → 최대 40자로 자름
- AI 응답도 URL 대신 인덱스 번호로 반환 (`"indices": [1, 2, 3]`)
- 서버에서 인덱스로 원본 item 매핑

### 인덱스 방식의 장점

- AI가 긴 URL을 응답에 그대로 출력 안 해도 됨 → 응답 토큰도 절약
- URL 오타/변형 없이 정확한 매핑 보장

---

## 문제 3: URL 필터링 보안

### 기존 방식

`startsWith("http")` 조건 → `chrome://` 북마크 누락, `javascript:` 북마크릿은 포함 가능

### 문제

- 악의적으로 만들어진 북마크릿(`javascript:`)이 프롬프트에 포함되면 프롬프트 인젝션 가능
- `data:` URL은 내용이 길어 토큰 낭비

### 해결: 화이트리스트 방식

```typescript
const ALLOWED_PROTOCOLS = ["http://", "https://", "chrome://", "file://"];
```

- 허용 프로토콜만 명시적으로 통과
- `javascript:`, `data:` 등은 차단
- `chrome://`, `file://`은 실제 사용 사례 있으므로 포함
