# Smart Bookmark — 북마크 저장 프로세스 & 엣지 케이스

## 전체 프로세스 개요

```
URL 입력
  ↓
크롤링 시도 (title + description + 본문 3등분)
  ├─ 성공 → 카드 표시 → AI 분석 → 써머리 + 태그 + 임베딩 저장
  └─ 실패 → 유저가 직접 입력 or 취소
              ├─ 입력함 → AI 분석 → 카드 저장
              └─ 취소 → 저장 안 됨
```

---

## DB 구조

```ts
{
  url,
  title,
  summary,        // AI 생성, 시맨틱 서치용
  embedding,      // summary 기반 벡터
  tags,           // AI 태그 + 유저 태그 (source: 'ai' | 'user')
  memo,           // 유저 메모, 단어 검색용
  isAiGenerated,  // 크롤링 실패 시 AI 생성 뱃지용
  crawlStatus,    // 'success' | 'partial' | 'failed' (내부용)
}
```

## 단계별 엣지 케이스

### 1. 크롤링 시작

| 케이스                              | 처리 방법                              |
| ----------------------------------- | -------------------------------------- |
| URL 형식이 잘못됨 (http 없음, 오타) | 입력 단계에서 validation               |
| 존재하지 않는 URL (404)             | 크롤링 실패 처리 → 유저 직접 입력 유도 |
| 접근 거부 (403, 로그인 필요)        | 크롤링 실패 처리 → 유저 직접 입력 유도 |
| 타임아웃 (서버 응답 없음)           | 최대 3회 재시도 후 실패 처리           |
| 중복 URL (이미 저장된 북마크)       | "이미 저장된 북마크입니다" 토스트      |

---

### 2. 카드에 결과 보여주기 (크롤링 성공 시)

| 케이스              | 처리 방법           |
| ------------------- | ------------------- |
| og:title 없음       | URL을 제목으로 대체 |
| og:image 없음       | 기본 이미지 표시    |
| og:description 없음 | 빈 상태로 표시      |
| 제목이 너무 긺      | `...` 처리          |

---

### 3. AI에 데이터 보내기

> title + description + 본문 3등분 샘플링 (각 300자씩, 총 900자)

| 케이스                             | 처리 방법                                     |
| ---------------------------------- | --------------------------------------------- |
| 본문이 아예 없음                   | title + description만 전송                    |
| 본문이 너무 짧음 (300자 미만)      | 3등분 의미 없음, 전체 그대로 전송             |
| 본문이 전부 이미지                 | 텍스트 추출 불가 → title + description만 전송 |
| AI 응답 지연 (타임아웃)            | 백그라운드 자동 3회 재시도                    |
| AI 응답 형식 오류 (JSON 파싱 실패) | 재시도 후 실패 시 카드에 재시도 버튼 표시     |
| AI API 한도 초과                   | 토스트 "잠시 후 다시 시도해주세요"            |

---

### 4. 써머리 생성

| 케이스                    | 처리 방법                                   |
| ------------------------- | ------------------------------------------- |
| 써머리가 너무 짧음        | 그대로 사용 (품질 이슈지만 UX 영향 없음)    |
| 써머리가 너무 긺          | 카드에서 `...` 처리                         |
| 언어가 한국어가 아닌 경우 | 해당 언어 그대로 사용                       |
| AI가 엉뚱한 내용 생성     | isAiGenerated 뱃지로 구분, 유저가 확인 가능 |

---

### 5. 임베딩 저장

| 케이스                         | 처리 방법                                     |
| ------------------------------ | --------------------------------------------- |
| 임베딩 API 실패                | 카드에 "AI 분석 재시도" 버튼 표시             |
| 써머리 없어서 임베딩 소스 없음 | 임베딩 저장 스킵, 단어 검색만 가능            |
| Supabase 저장 실패             | 토스트 "저장에 실패했어요. 다시 시도해주세요" |

---

### 6. 결과 보여주기

| 케이스                                 | 처리 방법                         |
| -------------------------------------- | --------------------------------- |
| 카드 렌더링 중 데이터 일부 없음        | 없는 필드는 빈 상태로 렌더링      |
| 동시에 여러 북마크 저장 중 순서 뒤바뀜 | 각 카드 독립적으로 로딩 상태 관리 |
| 저장 완료 전 유저가 페이지 이탈        | 백그라운드에서 계속 처리          |

## AI 분석 실패 처리 흐름

```
백그라운드 자동 3회 재시도
  ↓
그래도 실패
  ├─ 크롤링 성공했으면
  │    → title + description으로 카드 저장
  │    → 카드에 "AI 분석 재시도" 버튼 표시
  │    → 토스트 "AI 분석에 실패했어요"
  └─ 크롤링도 실패했으면
       → 저장 안 됨
```

```
"AI 분석 재시도" 버튼 클릭
  ├─ 성공 → 써머리 + 태그 + 임베딩 생성 → 버튼 사라짐
  └─ 실패 → 토스트 "나중에 다시 시도해주세요" → 버튼 유지
```

## 검색 프로세스

```
검색어 입력
  ↓
summary 시맨틱 서치 + title + memo 단어 검색 동시에
  ↓
점수 합산 → 하나의 리스트로 반환 (Hybrid Search)
```

### 데이터별 역할

| 필드    | 검색 방식                  |
| ------- | -------------------------- |
| summary | 시맨틱 서치 (임베딩)       |
| title   | 단어 검색                  |
| memo    | 단어 검색 (유저 개인 메모) |
| tags    | 단어 검색                  |

---

## AI 생성 뱃지 제거 조건

```
유저가 title or description 수정 후 저장
  ↓
기존 값과 비교
  ├─ 하나라도 바뀌었으면 → isAiGenerated = false (뱃지 제거)
  └─ 그대로면 → isAiGenerated 유지
```
