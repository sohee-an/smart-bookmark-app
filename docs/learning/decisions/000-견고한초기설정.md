# ADR 000: 견고한 프로젝트 초기 설정 (Environment & Quality Automation)

## 1. 개요 (Context)

다양한 개발 환경(OS, Node 버전 등)에서 발생할 수 있는 환경 차이를 제거하고, 코드 품질을 일관되게 유지하며, 휴먼 에러를 시스템적으로 방지하기 위해 강력한 제약 조건을 도입한다.

## 2. 설계 결정 (Design Decisions)

### 2.1. 런타임 및 패키지 매니저 고정 (Environment Isolation)

프로젝트의 안정성을 위해 개발 런타임 버전을 엄격히 강제한다.

#### **A. .nvmrc (Node 버전 가이드)**

팀원 간 Node.js 메이저 버전을 일치시키기 위해 사용한다. (Node 24.x 권장)

```bash
# .nvmrc
24
```

#### **B. .npmrc (엄격한 엔진 체크)**

지정된 버전이 아닐 경우 `pnpm install` 자체가 실패하도록 강제한다.

```ini
# .npmrc
engine-strict=true
```

#### **C. package.json (버전 명시)**

`engines` 필드와 `packageManager`를 통해 최소 요구 사항과 도구 버전을 고정한다.

```json
// package.json
{
  "engines": {
    "node": ">=24.0.0",
    "pnpm": ">=10.0.0"
  },
  "packageManager": "pnpm@10.30.2"
}
```

### 2.2. 코드 품질 및 휴먼 에러 방지 자동화 (Automation)

수동 리뷰 없이도 기본적인 코드 스타일과 품질이 보장되도록 도구를 구성한다.

#### **A. ESLint & Prettier (Static Analysis & Formatting)**

- **ESLint**: Next.js, TypeScript, Storybook 규칙 통합. Flat Config 사용.
- **Prettier**: `prettier-plugin-tailwindcss`를 통해 Tailwind 클래스 자동 정렬.

#### **B. Commitlint (커밋 메시지 규격화)**

`feat:`, `fix:`, `chore:` 등 Conventional Commits 규격을 강제하여 변경 이력의 가독성을 높인다.

```javascript
// commitlint.config.mjs (설정 예시)
export default { extends: ["@commitlint/config-conventional"] };
```

#### **C. Husky Hooks (Gatekeeping)**

실수를 원천 차단하기 위해 Git 훅을 활용한다.

- **pre-commit**: 변경된 파일만 Lint 및 Formatting 수행 (`lint-staged`).
- **commit-msg**: 커밋 메시지 규격 검사 (`commitlint`).
- **pre-push**: 푸시 전 전체 프로젝트 타입 체크 (`tsc --noEmit`).

```bash
# .husky/pre-push 예시
pnpm exec tsc --noEmit
```

#### **D. Storybook (UI Isolation)**

- UI 컴포넌트를 비즈니스 로직과 분리하여 독립 개발 및 문서화.

### 2.3. 모노레포(Monorepo) 아키텍처 채택

FSD(Feature-Sliced Design) 패턴의 효율성을 극대화하기 위해 `pnpm workspace` 기반의 모노레포 구조로 전환한다.

- **네임스페이스**: `@sb/` (Smart Bookmark) 사용.
- **패키지 분리**: `@sb/ui`, `@sb/shared`, `@sb/entities`, `@sb/configs`.

## 3. 판단 기준 및 기대 효과 (Rationale)

1.  **환경 일관성**: `.npmrc`의 `engine-strict` 덕분에 잘못된 버전에서의 작업 가능성을 원천 차단한다.
2.  **기술 부채 방지**: `pre-commit`과 `pre-push`를 통해 문법 오류나 타입 에러가 저장소에 유입되는 것을 방지한다.
3.  **의사소통 효율**: `Commitlint`를 통해 커밋 메시지만 보고도 변경 사항의 성격(기능 추가, 버그 수정 등)을 즉시 파악할 수 있다.

## 4. 향후 과제

- `apps/` 및 `packages/` 폴더 구조 실제 생성 및 코드 마이그레이션.
- 공통 설정을 `packages/configs`로 추출하여 전역 패키지 관리.
