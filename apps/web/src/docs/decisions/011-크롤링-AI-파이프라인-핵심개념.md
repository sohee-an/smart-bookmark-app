# 011 - 크롤링 → AI 파이프라인 구현 핵심 개념

## 개요

북마크 저장 시 URL 크롤링 → AI 분석까지의 파이프라인을 구현하면서 익혀야 할 핵심 개념들을 정리한다.

---

## 1. 낙관적 업데이트 (Optimistic Update)

서버 응답을 기다리지 않고 UI를 먼저 업데이트하는 패턴.
사용자는 즉각적인 피드백을 받고, 백그라운드에서 실제 작업이 진행된다.

```typescript
// AddBookmarkOverlay.tsx

const handleSave = async (e: React.FormEvent) => {
  // 1. DB 저장
  const newBookmark = await bookmarkService.addBookmark(url, memo);

  // 2. 서버 응답을 기다리지 않고 즉시 UI 반영 ← 낙관적 업데이트
  addBookmark(newBookmark); // aiStatus: "crawling" 상태로 카드 즉시 표시
  handleClose();            // 모달 바로 닫음

  // 3. 백그라운드에서 파이프라인 진행 (UI는 이미 업데이트됨)
  const crawlRes = await fetch("/api/crawl", { ... });
  // ...
};
```

실패 시에는 aiStatus를 "failed"로 전환해 에러 상태를 표시한다.

---

## 2. 파이프라인 단계별 상태 전환

하나의 긴 작업을 단계별로 쪼개 중간 결과를 보여주는 설계.
단계마다 `aiStatus`를 업데이트해 카드 UI가 실시간으로 바뀐다.

```
crawling   → 크롤링 중 (전체 스켈레톤)
processing → AI 분석 중 (썸네일/제목 표시, summary 자리에 로딩)
completed  → 완료 (전부 표시)
failed     → 실패 (에러 메시지)
```

```typescript
// types.ts
export type AIStatus = "crawling" | "processing" | "completed" | "failed";
```

```typescript
// 파이프라인 흐름 (AddBookmarkOverlay.tsx)

// 1단계: 저장 직후 → crawling
const newBookmark = await bookmarkService.addBookmark(url); // aiStatus: "crawling"
addBookmark(newBookmark);

// 2단계: 크롤링 완료 → processing
const { title, thumbnailUrl, description, bodyChunks } = crawlJson.data;
const crawlUpdate = { thumbnailUrl, aiStatus: "processing", ...(title ? { title } : {}) };
updateBookmark(bookmarkId, crawlUpdate);

// 3단계: AI 완료 → completed
const { title: aiTitle, summary, tags } = aiJson.data;
const finalUpdate = {
  summary,
  tags,
  aiStatus: "completed",
  ...(aiTitle ? { title: aiTitle } : {}),
};
updateBookmark(bookmarkId, finalUpdate);
```

기존 `process-url.ts`처럼 한 번에 처리하면 중간 상태를 보여줄 수 없다.
단계를 나눠야 UI가 단계마다 반응한다.

## 3. Next.js `pages/api/` 규칙

`pages/api/` 하위 모든 파일이 API Route로 간주된다.

```
pages/api/crawl.ts         → /api/crawl       ✅ 의도한 Route
pages/api/ai-analyze.ts    → /api/ai-analyze   ✅ 의도한 Route
pages/api/services/ai.ts   → /api/services/ai  ❌ 서비스 파일인데 Route로 인식
```

서비스 파일을 `pages/api/` 안에 두면 빌드 시 아래 에러가 발생한다.

```
Type error: Property 'default' is missing
→ API Route는 default export handler 함수가 필수
```

**해결책**: 서비스 로직을 `pages/api/` 밖으로 분리 (`src/server/services/`)
또는 로직을 API Route 파일 안에 직접 작성한다.

```typescript
// pages/api/ai-analyze.ts — 올바른 구조
export default async function handler(req: NextApiRequest, res: NextApiResponse) {
  // Gemini 로직 직접 작성
}
```

## 4. 서버/클라이언트 경계

실행 환경은 **파일 위치가 아니라 누가 import하느냐**가 결정한다.

```
브라우저
  ↓ fetch('/api/ai-analyze')
pages/api/ai-analyze.ts  ← Next.js 서버에서 실행
  ↓ (내부 로직)
GEMINI_API_KEY 사용      ← 브라우저에 절대 노출 안 됨
```

| 서버에서 해야 하는 이유 | 해결책                    |
| ----------------------- | ------------------------- |
| API 키 노출 방지        | API Route 안에서만 사용   |
| CORS 우회               | 서버가 외부 URL 직접 요청 |

```typescript
// ❌ 클라이언트에서 직접 호출 → API 키 노출
const genAI = new GoogleGenerativeAI(process.env.GEMINI_API_KEY); // 브라우저에 노출됨

// ✅ API Route를 통해 간접 호출 → 키 보호
const res = await fetch('/api/ai-analyze', { method: 'POST', body: ... });
```

## 5. Zustand 전역 상태 부분 업데이트

백그라운드 작업 결과를 받을 때마다 해당 카드만 업데이트하는 패턴.

```typescript
// useBookmarkStore.ts
updateBookmark: (id, data) =>
  set((state) => ({
    bookmarks: state.bookmarks.map((b) =>
      b.id === id ? { ...b, ...data } : b  // 해당 ID만 병합
    ),
  })),
```

```typescript
// 사용 예시 — 크롤링 결과 부분 업데이트
updateBookmark(bookmarkId, { title, thumbnailUrl, aiStatus: "processing" });

// AI 결과 부분 업데이트
updateBookmark(bookmarkId, { summary, tags, aiStatus: "completed" });
```

`{ ...b, ...data }` 스프레드로 기존 필드는 유지하고 변경된 필드만 덮어쓴다.
React는 변경된 해당 컴포넌트만 자동으로 리렌더링한다.

## 6. title이 없을 때 AI로 생성하는 전략

크롤링 시 title을 못 가져온 경우 본문(bodyChunks)을 AI에게 넘겨 생성시킨다.

```typescript
// crawler.service.ts — bodyChunks를 title 체크 전에 먼저 추출
const bodyText = $("p, article, main").text().replace(/\s+/g, " ").trim();
let bodyChunks: [string, string, string] | undefined;
if (bodyText.length > 0) {
  const chunkSize = Math.ceil(bodyText.length / 3);
  bodyChunks = [
    bodyText.slice(0, chunkSize),
    bodyText.slice(chunkSize, chunkSize * 2),
    bodyText.slice(chunkSize * 2),
  ];
}

// title 없어도 bodyChunks 있으면 AI가 생성 가능 → success 반환
if (!title && !bodyChunks) {
  return this.handleRetry(url, attempt, "TITLE_NOT_FOUND"); // 둘 다 없을 때만 실패
}
```

```typescript
// ai-analyze.ts — AI 프롬프트에서 title 생성 조건 명시
const prompt = `
제목: ${title || "(없음)"}
본문: ${bodyText.slice(0, 2000)}

- title이 "(없음)"이면 본문을 보고 제목을 생성하고, 있으면 null을 반환하세요.
- 형식: { "title": "생성된제목 또는 null", "summary": "...", "tags": [...] }
`;
```

```typescript
// AddBookmarkOverlay.tsx — AI가 생성한 title이 있을 때만 반영
const { title: aiTitle, summary, tags } = aiJson.data;
const finalUpdate = {
  summary,
  tags,
  aiStatus: "completed" as const,
  ...(aiTitle ? { title: aiTitle } : {}), // null이면 기존 title 유지
};
updateBookmark(bookmarkId, finalUpdate);
```
